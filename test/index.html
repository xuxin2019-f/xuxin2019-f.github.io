# 1.伪类和伪元素

面试题：

  1. ::before 和 :after 中双冒号和单冒号 有什么区别？解释一下这 2 个伪元素的作用
  • 单冒号(:)用于CSS3伪类，双冒号(::)用于CSS3伪元素
  • 用于区分伪类和伪元素

2.伪类和伪元素的区别

伪类表状态；伪元素是真的有元素；伪类是单冒号，伪元素是双冒号

伪类：:hover :visited :active :linked :focus

​ :nth-child() :first-child :last-child :nth-of-type() :nth-last-of-type() （注意这些都是作用在子元素身上的，通过子元素找到父元素然后检索该父元素的所有子元素）

伪元素：之所以被称为伪元素，是因为他们不是真正的页面元素，html 没有对应的元素，但是其所有用法和表现行为与真正的页面元素一样，可以对其使用诸如页面元素一样的 css 样式，表面上看上去貌似是页面的某些元素来展现，实际上是 css 样式展现的行为，因此被称为伪元素。

重要有::befor,::after用法详解

伪元素妙用

应用场景：清除浮动

.clearfix {
  zoom: 1; /*兼容IE浏览器6、7低版本*/
}

.clearfix::after {
  content: '';
  height: 0;
  line-height: 0;
  display: block;
  clear: both;
  visibility: hidden; /*将元素隐藏起来*/
  /*在页面的clearfix元素后面添加了一个空的块级元素，这个元素清除了浮动*/
}

制作特效

这两个伪元素分别表示元素内容的【前】【后】，利用这两个伪元素可以在元素内容的前后添加内容，其实这没有什么前后的概念，如果应用了 absolute 的特性之后，你可以把这些伪元素放在任何位置，有了这两个伪元素，就代表每个元素都有两个助手可供使用，灵活运用它们的话将会得到很多有趣的实现，简化许多实现。

# 2.css 盒模型

# 1.w3c 的标准盒子模型

css 盒模型用来封装所有的 html 元素，它包括外边距 margin，边框 border，内边距 padding，和实际内容 content

当指定一个 CSS 元素的宽度和高度属性时，你只是设置内容区域的宽度和高度。要知道，完全大小的元素，你还必须添加填充，边框和边距。.

css 盒模型的实际宽度=设定宽度+左右外边距+左右边框+左右内边距

css 盒模型的实际高度=设定高度+上下外边距+上下边框+上下内边距

# 2.IE 标准的盒子模型（怪异盒模型）

在 ie9 下自动触发

怪异盒模型中的width指的是内容、边框、内边距总的宽度（content + border + padding）；height指的是内容、边框、内边距总的高度

怪异盒模型下盒子的大小=width（content + border + padding） + margin

# 3.使用 box-sizing 设置盒模型

我们还可以通过属性 box-sizing 来设置盒子模型的解析模式

可以为box-sizing赋三个值：

content-box： 默认值，border 和 padding 不算到 width 范围内，可以理解为是 W3c 的标准模型(default)

border-box：border 和 padding 划归到 width 范围内，可以理解为是 IE 的怪异盒模型

**padding-box：**将 padding 算入 width 范围

  • 当设置为 box-sizing:content-box 时，将采用标准模式解析计算（默认模式）；

  • 当设置为 box-sizing:border-box 时，将采用怪异模式解析计算；

# border-box 应用场景

当我们希望通过百分比设定整个盒子的宽高，而不是内容宽高时，或者直接希望子元素 dom 完全撑满父元素时，需要用到 border-box。因为默认的 content-box 会使 padding 和 border 在原有的基础上溢出

# 4.CSS 优先级

  • 优先级就近原则，同权重情况下样式定义最近者为准
  • 载入样式以最后载入的定位为准
  • 优先级为: !important > 内联样式>id > class > tag; !important 比 内联优先级高
  • ！important规则最重要，大于其它规则
  • 行内样式规则，加1000
  • 对于选择器中给定的各个ID属性值，加100
  • 对于选择器中给定的各个类属性、属性选择器或者伪类选择器，加10
  • 对于选择其中给定的各个元素标签选择器，加 1
  • 如果权值一样，则按照样式规则的先后顺序来应用，顺序靠后的覆盖靠前的规则

# 6.CSS 的性能优化

1.内联首屏的关键 CSS（控制大小小于 14.6kB：因为内联 css）

大家应该都习惯于通过 link 标签引用外部 CSS 文件。但需要知道的是，将 CSS 直接内联到 HTML 文档中能使 CSS 更快速地下载。而使用外部 CSS 文件时，需要在 HTML 文档下载完成后才知道所要引用的 CSS 文件，然后才下载它们。所以说，内联 CSS 能够使浏览器开始页面渲染的时间提前，因为在 HTML 下载完成之后就能渲染了。

2.合并：每个声明只使用一次

严格使用选择器组合。

例如，你可以将这两个规则

  1. h1 { color: black; }
  2. p { color: black;

合并成一个规则

  1. h1, p { color: black; }

3.使用 gzip 压缩页面（现在的构建工具，如 webpack、gulp/grunt、rollup 等也都支持 CSS 压缩功能）

Gzip 压缩需要 Web 服务器的支持，在没有 gzip 压缩的情况下，Web 服务器直接把 html 页面发送给浏览器，而支持 gzip 的 Web 服务器将把 html 压缩后再发给浏览器，支持 gzip 的浏览器再在本地进行解压和解码，并显示出来。Gizp 会将 css 文件，js 文件，html 文件中那些重复的字符串、标签、空格和样式定义进行压缩。

4.不要使用@import @import 过多嵌套会导致串行下载，影响浏览器的并行下载，出现 FOUC（文档样式短暂失效）

多个@import 会导致下载顺序紊乱

结论：使用 link 标签放在

来代替使用@import

# 7.stylus/sass/less 区别

都是 css 的扩展语言，都支持嵌套的语法。

  • sass 和 less 都是用标准的 css 语法，stylus 支持的语法更多样性
  • sass 基于 ruby，less 基于 js
  • 都引入了变量的概念，sass 中变量以$开头，less以@开头，stylus可以以$或除了@以外的任何字符开头

# 8.什么是外边距重叠？重叠的结果是什么

即 margin-collapse

折叠结果遵循下列计算规则：

  • 两个相邻的外边距都是正数时，折叠结果是它们两者之间较大的值。
  • 两个相邻的外边距都是负数时，折叠结果是两者绝对值的较大值。
  • 两个外边距一正一负时，折叠结果是两者的相加的和。

# 9.rgba()和 opacity 的透明效果有什么不同？

  • rgba()和opacity都能实现透明效果，但最大的不同是opacity作用于元素，以及元素内的所有内容的透明度，
  • 而rgba()只作用于元素的颜色或其背景色。（设置rgba透明的元素的子元素不会继承透明效果！）

# 11.水平居中与垂直居中

  • 水平居中

​ 1. 已知宽度的情况下： margin-left：父元素宽度/2 + transform:translateX(-50%)

​ 2. 父元素设置 flex,flex-direction 为 row，justify-content 为 center

​ 3. 设置子元素为 inline - block，然后父元素利用 text-align 设置行内元素的水平居中

​ 4. 父相子绝：left : 50%+ transform:translateX(-50%)

​ 5. margin : 0 auto ;

  • 垂直居中
  1. 已知高度的情况下同上 1
  2. 父元素设置 flex，flex-direction 为 column，单个子元素设置 align-self：center
  3. 父元素设置 flex，flex-direction 为 column，并设置 align-items：center
  4. 父元素设置 display：table-cell+verticle-align:middle（表格单元格是一行）
  5. 在父元素里添加一个空 div，高度设为 100%，宽度设为 0，将这个 div 和子元素都设置为 inline-block（设置为同一行保证基准线的一致），然后将这个 div 和子元素都设置 verticle-align:middle
  6. 垂直居中的单行行内元素，直接把子元素的行高设置为高度即可 ： line-height :200px ; height:200px;
  7. 父相子绝：子元素设置 top:50% + transform:translateY(-50%)

注意：verticle-align 这个属性不是单纯的指垂直居中，它只作用于在同一行内的元素，该属性定义行内元素的基线相对于该元素所在行的基线的垂直对齐。因此要么搭配 table-cell，要么设置一个高度等于父元素的空元素保证与子元素是一行。

# 12.重绘和回流（重排）是什么，如何避免

重排（reflow)指对 dom 树进行渲染，只要修改 dom 或者修改元素的形状大小，就会触发 reflow

重绘(repaint)指当我们对 dom 的修改导致样式变化，却没有影响几何属性时，浏览器不需要重新计算元素的几何属性，直接可以为该元素绘制新的样式

引起重排的原因：

  • 添加或者删除可见的 DOM 元素，
  • 元素位置、尺寸、内容改变，
  • 浏览器页面初始化，
  • 浏览器窗口尺寸改变，重排一定重绘，重绘不一定重排

注意：

由于 display:none 后元素不占空间了，所以会引起重排；而 visibility：hidden 后元素仍然占据原来的空间，并不会引起重排，只是重绘

如何避免：

1.在布局信息改变时不做 dom 操作

2.用 className 或 cssText 一次性改变属性

cssText：https://www.cnblogs.com/majingyi/p/6840818.html

通常用 js 来复写对象样式，当样式多的时候，语句多，会造成多次的重排，比如：

var element= document.getElementById(“id”);
element.style.width=”20px”;
element.style.height=”20px”;
element.style.border=”solid 1px red”;

js 中有一个 cssText 的方法，语法为element.style.cssText=”width:20px;height:20px;border:solid 1px red;”;

可以尽量避免页面 reflow，提高性能

className：https://www.cnblogs.com/GreenLeaves/p/5757216.html

css： .change {
  background-color: red;
}

btn2.addEventListener('click', function () {
  // 用className
  box.className = 'change'
})

3.使用 fragment 语法：

let fragment = document.createDocumentFragment();

fragment 是一个指向空DocumentFragment对象的引用。

DocumentFragments 是 DOM 节点。它们不是主 DOM 树的一部分。通常的用例是创建文档片段，将元素附加到文档片段，然后将文档片段附加到 DOM 树。在 DOM 树中，文档片段被其所有的子元素所代替。

因为文档片段存在于内存中，并不在 DOM 树中，所以将子元素插入到文档片段时不会引起页面回流（对元素位置和几何上的计算）。因此，使用文档片段通常会带来更好的性能。

4.对于多次重排的元素，比如动画，使用绝对定位使其脱离文档流，让它的改变不影响其他元素

# 13.css 中使元素脱离文档流的方法

  • 浮动
  • 绝对定位

所谓绝对定位即 position:absolute

绝对定位之后,标签就不区分所谓的行内元素,块级元素了,不需要display:block;就可以设置宽高了

所谓相对定位即 position:relative（并不脱离文档流）

一般实现绝对定位的搭配是子绝父相：子绝父绝,子绝父相,子绝父固,都是可以给儿子定位的.但是,工程上,子绝,父绝,没有一个盒子在标准文档流中,所以页面就不稳固,没有任何实战用途.工程上,"子绝父相"有意义,父亲没有脱标,儿子脱标在父亲的范围里面移动.

注意！绝对定位的参考点：如果用 top 描述，则是页面的左上角，而不是浏览器的左上角

如果是 bottom 描述，则是浏览器首屏窗口尺寸对应的页面的左下角

# 14.什么是 BFC

BFC 是块级格式化上下文，是一个独立的渲染区域，它内部的子元素如何布局与区域外部毫不相关。

如何创建 BFC:

  • 根元素

  • float 不为 null

  • display 为 table-cell、inline-block、flex、table-caption 或 inline-flex

  • position 为 absolute 或 fixed

  • overflow 的值不是 visible 的

BFC 的布局规则：

  • 内部的 Box 会在垂直方向，一个接一个地放置。
  • Box 垂直方向的距离由 margin 决定。属于同一个BFC 的两个相邻 Box 的 margin 会发生重叠。（衍生解决 margin 塌陷的问题：为另一个子元素生成一个独立的 BFC）
  • 每个盒子（块盒与行盒）的 margin box 的左边，与包含块 border box 的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。（解决方法也是每个盒子生成独立的 BFC)
  • BFC 的区域不会与 float box 重叠。
  • BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算 BFC 的高度时，浮动元素也参与计算。(用于清除浮动：将父元素变为 BFC，这样即使父元素没有高度，且子元素是浮动元素时，将父元素设为 BFC，就会自动将父元素撑开)

BFC 的应用场景：

  • 利用 BFC 避免 margin 重叠。
<style>
  * {
    margin: 0;
    padding: 0;
  }
  p {
    color: #f55;
    background: yellow;
    width: 200px;
    line-height: 100px;
    text-align: center;
    margin: 30px;
  }
</style>
<body>
  <p>看看我的 margin是多少</p>
  <p>看看我的 margin是多少</p>
</body>

此时两个 p 属于同一个 BFC,而属于同一个 BFC 的两个相邻的 Box 会发生 margin 重叠，所以我们可以设置，两个不同的 BFC，也就是我们可以让把第二个 p 用 div 包起来，然后激活它使其成为一个 BFC

<body>
  <p>看看我的 margin是多少</p>
  <div>
    <p>看看我的 margin是多少</p>
  </div>
</body>
  • 自适应两栏布局

由于每个盒子的 margin box 的左边，与包含块 border box 的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。

<style>
  * {
    margin: 0;
    padding: 0;
  }
  body {
    width: 100%;
    position: relative;
  }

  .left {
    width: 100px;
    height: 150px;
    float: left;
    background: rgb(139, 214, 78);
    text-align: center;
    line-height: 150px;
    font-size: 20px;
  }

  .right {
    height: 300px;
    background: rgb(170, 54, 236);
    text-align: center;
    line-height: 300px;
    font-size: 40px;
  }
</style>
<body>
  <div class="left">LEFT</div>
  <div class="right">RIGHT</div>
</body>

效果：

又因为 BFC 区域不会和 float box 重叠，因此将 right 设置为一个单独的 BFC 就可以避免与 left 重叠了：设置 right 属性：overflow：hidden

  • 清除浮动

当我们不给父节点设置高度，子节点设置浮动的时候，会发生高度塌陷，这个时候我们就要清楚浮动。

而根据：计算 BFC 的高度时，浮动元素也参与计算。给父元素激活成 BFC，就可以清除浮动。

<style>
			.par{
				background-color: #00FFFF;
				border: 3px solid red;
				/* 为了清除浮动，将父元素设为BFC */
				overflow: hidden;
			}
			.child{
				width: 100px;
				height: 200px;
				border: 5px solid yellow;
				float: left;
			}
		</style>
	</head>
	<body>
		<div class="par">
			<div class="child"></div>
			<div class="child"></div>
		</div>

# 16.层叠上下文、层叠等级、层叠顺序、z-index

https://juejin.im/post/5b876f86518825431079ddd6#heading-2

注意：

  1. z-index 只在设定了 position 非 static 的定位元素上有效果

  2. 判断元素在 z 轴上的堆叠顺序，不仅仅是直接比较两个元素的 z-index，还要由元素的层叠上下文、层叠等级共同决定。

层叠上下文

如果一个元素含有层叠上下文，(也就是说它是层叠上下文元素)，我们可以理解为这个元素在Z轴上就“高人一等”，最终表现就是它离屏幕观察者更近。

如何实现一个层叠上下文：

  1. HTML中的根元素``本身 j 就具有层叠上下文，称为“根层叠上下文”。

  2. 普通元素设置position属性为非static值并设置z-index属性为具体数值，产生层叠上下文。

  3. CSS3 中的新属性也可以产生层叠上下文：

    父元素的 display 属性值为flex|inline-flex，子元素z-index属性值不为auto的时候，子元素为层叠上下文元素；

    元素的opacity属性值不是1；

    元素的transform属性值不是none；

    元素mix-blend-mode属性值不是normal`；

    元素的filter属性值不是none；

    元素的isolation属性值是isolate；

    will-change指定的属性值为上面任意一个；

    元素的-webkit-overflow-scrolling属性值设置为touch。

层叠等级

  • 在同一个层叠上下文中，它描述定义的是该层叠上下文中的层叠上下文元素在Z轴上的上下顺序。
  • 在其他普通元素中，它描述定义的是这些普通元素在Z轴上的上下顺序。

层叠等级的比较只有在当前层叠上下文元素中才有意义。不同层叠上下文中比较层叠等级是没有意义的。
